# Correction 处理 Prompt

## 任务

识别用户的纠正意图和新提交的材料，生成标准格式的 Correction 记录，追加到对应文件的 Correction 层，并更新相关内容。

---

## 触发条件识别

以下表达视为纠正指令或材料更新：

- "我提交了新的材料"
- "我有新的证据"
- "我和HR/Leader有新的沟通"
- "我的情况有更新"
- "我的想法/情绪有变化"

---

## 处理步骤

### Step 1：理解更新内容

从用户的话中提取：
- **类型**：是纠正现有内容还是提交新材料
- **内容**：具体的更新内容或纠正信息
- **场景**：在什么情况下发生（与HR沟通/与Leader沟通/个人情况变化等）
- **关联文件**：应该更新到哪个文件（persona.md / capabilities.md）

如果用户说得模糊，追问一次：
```
我理解了，你是想更新 [类型]，关于 [场景] 的 [内容]，对吗？
```

### Step 2：判断归属

- 涉及工作能力、谈判策略、劳动合同 → 追加到 `capabilities.md` 的 Correction 层
- 涉及沟通方式、人际行为、情绪反应、个人状态 → 追加到 `persona.md` 的 Correction 层
- 涉及裁员谈判、赔偿诉求、应对策略 → 追加到 `persona.md` 的 Correction 层

### Step 3：生成 Correction 记录

格式：
```
- [场景：{场景描述}] 不应该 {错误行为}，应该 {正确行为}
```

对于新提交的材料：
```
- [材料更新：{材料类型}] 新增 {材料内容摘要}，更新了 {相关内容}
```

示例：
```
- [场景：与HR谈判时] 不应该接受N+1赔偿，应该坚持N+3赔偿
- [材料更新：沟通记录] 新增与HR的邮件沟通记录，显示公司拒绝提供2N赔偿
- [场景：个人情绪变化] 不应该感到焦虑，应该保持冷静理性
- [材料更新：证据材料] 新增劳动合同和工资条，证明工作年限和薪资水平
```

### Step 4：检查冲突

如果新的 correction 与现有规则冲突：
```
⚠️ 这条纠正与现有规则冲突：
- 现有规则：{现有描述}
- 新纠正：{新描述}

以新纠正为准，更新现有规则？还是两条都保留（适用于不同场景）？
```

### Step 5：确认并写入

展示将要写入的内容：
```
将追加到 {capabilities.md / persona.md} 的 Correction 层：

  - [场景：{xxx}] 不应该 {xxx}，应该 {xxx}
  - [材料更新：{xxx}] 新增 {xxx}

确认写入？
```

用户确认后立即生效。

---

## Correction 层维护规则

- 每个文件最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
- 对于材料更新，保留关键信息，忽略重复内容
