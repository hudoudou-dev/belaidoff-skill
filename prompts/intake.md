# 个人情况录入脚本

## 开场白

```
我来帮你创建被裁员应对 Skill。只需要回答几个问题，每个都可以跳过。
```

---

## 问题序列

### Q1：姓名/昵称

```
你怎么称呼？（姓名、昵称或代号都行，多个字用 - 连接）

例：张三，zhangsan
```

- 接受任意字符串
- 生成的 slug 统一用 `-` 连接（不用下划线）
- 中文自动转小写拼音（"张三" → `zhangsan`，"小李" → `xiaoli`）
- 英文直接转小写连接（"John Doe" → `johndoe`）

---

### Q2：工作基本信息

把公司、入职时间、职级、年薪、绩效情况放在一个问题里，让用户一句话说完：

```
用一句话描述你的工作情况——公司、入职时间、职级、年薪、绩效情况，想到什么写什么，跳过也行。

例：阿里巴巴 2020-01-01 13 60万/年 325
```

从用户的回答中解析以下字段（缺失的留空）：
- **公司**：如 阿里巴巴/腾讯/百度/美团/字节跳动/字节/动
- **入职时间**：如 2020-01-01
- **职级**：如 P7 / 3-1 / T3-2 / L6 / 高级
- **年薪**：如 60万/年 或 25000/月
- **绩效情况**：如 A/B/C/优秀/良好/一般/325/350/375

### Q3：个人状态

把年龄、性别、学历、婚姻状况、子女情况、房贷情况放在一个问题里，让用户一句话说完：

```
用一句话描述你的个人状态——年龄、性别、学历、婚姻状况、子女情况、房贷情况，想到什么写什么，跳过也行。

例：32 硕士 已婚 1个 有（月供8000元）
```

从用户的回答中解析以下字段（缺失的留空）：
- **年龄**：如 35
- **性别**：如 男/女
- **学历**：如 本科/硕士/博士
- **婚姻状况**：如 未婚/已婚/离异
- **子女情况**：如 无/1个/2个
- **房贷情况**：如 无/有（月供X元）

---

### Q4：裁员情况

把裁员阶段和诉求放在一个问题里，让用户一句话说完：

```
用一句话描述你的裁员情况——当前阶段和诉求，想到什么写什么，跳过也行。

例：正在谈判 2N赔偿
```

从用户的回答中解析以下字段（缺失的留空）：
- **裁员阶段**：如 口头通知/书面通知/正在谈判/已离职
- **诉求**：如 2N赔偿/N+1赔偿/N+3赔偿/恢复工作/留用/转岗

---

### Q5：个人状态标签

让用户选择或描述自己的状态：

```
请描述你的状态（可选）：
- 家庭负担：无负担 / 有子女 / 有房贷 / 上有老下有小
- 心理状态：冷静理性 / 焦虑不安 / 愤怒维权 / 迷茫无助
- 沟通风格：直接强硬 / 委婉协商 / 沉默被动 / 据理力争
- 风险承受能力：低风险（希望尽快解决） / 中风险（可接受协商） / 高风险（坚持维权到底）

例：有子女 有房贷 焦虑不安 据理力争 中风险
```

从用户的回答中识别并提取以下标签（缺失的留空）：
- **家庭负担**
- **心理状态**
- **沟通风格**
- **风险承受能力**

## 确认汇总

收集完毕后展示：

```
信息汇总：

  👤  {姓名}
  🏢  {公司} {职级}，入职于 {入职时间}（若未填则省略）
  💰  {年薪}，绩效 {绩效情况}（若未填则省略）
  🧑‍🤝‍🧑  {年龄}岁，{性别}，{学历}，{婚姻状况}，{子女情况}，{父母情况}，{房贷情况}（若未填则省略）
  📋  裁员阶段：{裁员阶段}，诉求：{诉求}（若未填则省略）
  🏷️  状态标签：{标签列表}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```

用户确认后进入 Step 2 证据材料导入。
