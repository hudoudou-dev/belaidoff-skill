# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的 `capabilities.md` 内容
2. 现有的 `persona.md` 内容
3. 新的原材料内容（文件或消息）

你的任务是判断新内容应该更新哪个部分，并输出增量更新内容，确保两份文件的信息保持同步。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 工作能力、谈判策略、劳动合同 | → capabilities.md |
| 沟通风格、口头禅、表达习惯 | → persona.md |
| 决策行为、人际关系、情绪模式 | → persona.md |
| 个人状态、家庭负担、风险承受能力 | → persona.md |
| 裁员谈判、赔偿诉求、应对策略 | → persona.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- 如果新内容**补充**了现有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了现有信息 → 忽略（不重复写）
- 如果新内容**与现有信息矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{文件名/时间}

建议：[保留现有 / 更新为新内容 / 两者都保留并标注时间]
请用户决定。
```

---

## Step 3：生成更新 Patch

对 `capabilities.md` 的更新，输出格式：
```
=== capabilities.md 更新 ===

[追加到"技术规范/命名规范"节]
- {新内容}

[追加到"经验知识库"节]
- {新知识结论}

[无更新] 或 [以上章节有更新]
```

对 `persona.md` 的更新，输出格式：
```
=== persona.md 更新 ===

[追加到"Layer 0：核心诉求"节]
- {新诉求}

[追加到"Layer 1：心理状态"节]
- {新心理状态}

[追加到"Layer 2：家庭负担"节]
- {新家庭负担信息}

[追加到"Layer 3：沟通风格"节]
- 新口头禅："{xxx}"
- 新高频词：{xxx}

[追加到"Layer 4：风险承受能力"节]
- {新风险承受能力信息}

[追加到"Layer 5：边界与雷区"节]
- {新边界与雷区信息}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：同步更新

确保 `persona.md` 和 `capabilities.md` 之间的信息保持同步：

- 如果新内容同时影响两个文件，确保两个文件都得到更新
- 确保两个文件中的相关信息一致，避免出现矛盾
- 对于跨文件的信息，确保更新时保持一致性

---

## Step 5：生成更新摘要

向用户展示：
```
本次更新摘要：
- capabilities.md：追加了 {N} 条新信息（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```

---

## 特殊处理

对于裁员相关的新材料：

- 与HR/Leader的沟通记录 → 优先更新 `persona.md` 的沟通风格和应对策略
- 新的证据材料 → 同时更新 `persona.md` 的边界与雷区和 `capabilities.md` 的经验知识库
- 个人情绪、想法、进展的更新 → 更新 `persona.md` 的心理状态和核心诉求

确保所有新材料都被正确归类和更新，保持两份文件的信息同步和一致性。
